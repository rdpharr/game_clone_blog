# Building a 3D Mobile Runner Game

I'm creating a clone of Total Battle's runner/auto-battle mechanics using **Godot 4** as a solo developer with **Claude Code** as my coding assistant. This blog chronicles the journey from concept to Android APK.

**Important Note:** This is a **personal learning project for individual use only**. I'm not creating a commercial product or competing with the original game. This is about learning game development by recreating mechanics I find interesting, without any of the monetization pressure.

## Project Stats

**Engine:** Godot 4  
**Platform:** Android  
**Budget:** $0 (free assets only)  
**Timeline:** 3-5 months (Dec 2025 - April/May 2026)  
**Status:** Pre-Development (Planning Complete)  
**Purpose:** Learning & Personal Use Only

## Why Build This?

**The Real Motivation:**

I enjoy the core gameplay loop of Total Battle's runner mode, but I don't enjoy:
- Constant purchase prompts and paywalls
- Energy systems that limit playtime
- Aggressive monetization dark patterns
- Pay-to-win mechanics
- Daily login pressure and FOMO tactics

**What I want instead:**
- Pure gameplay without interruptions
- Play as much as I want, whenever I want
- No ads, no timers, no artificial restrictions
- Just the fun runner/combat mechanics
- A game I built myself, understanding every system

This project is about **reclaiming the joy of gaming** by building exactly what I want to play, free from all the psychological manipulation common in mobile games.

## What I'm Building

A clean, no-pressure 3D mobile runner combining:
- Auto-advancing lane-based movement
- Auto-shooting mechanics (target barrels and gates)
- Collectible multiplier zones
- Simple auto-battle combat
- Progressive difficulty across 20+ stages

Think Temple Run meets auto-chess, with a medieval fantasy theme â€” **but completely free of dark patterns**.

## The Tech Stack

**Why Godot 4?** 
- 100% free and open source
- Excellent 3D engine for mobile
- GDScript is perfect for AI-assisted development
- One-click Android export
- Zero hidden costs
- No revenue sharing or licensing fees

**Development Approach:**
- Build-first learning (minimal tutorials)
- Using Claude Code for guidance and code generation
- Free assets from Kenney.nl
- Git for version control
- Obsidian vault for documentation

## Development Timeline

### Weeks 1-2: Core Prototype

### Weeks 3-4: Advanced Mechanics  
Auto-shooting system, gate mechanics, multiplier zones, particle effects.

### Weeks 5-8: Content & Polish
Multiple unit types, enemy variety, visual effects, sound design.
