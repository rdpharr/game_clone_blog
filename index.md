# Building a 3D Mobile Runner Game

I'm creating a clone of Total Battle's runner/auto-battle mechanics using **Godot 4** as a solo developer with **Claude Code** as my coding assistant. This blog chronicles the journey from concept to Android APK.

## Project Stats

**Engine:** Godot 4  
**Platform:** Android  
**Budget:** $0 (free assets only)  
**Timeline:** 3-5 months (Dec 2025 - April/May 2026)  
**Status:** Pre-Development (Planning Complete)

## What I'm Building

A 3D mobile runner combining:
- Auto-advancing lane-based movement
- Auto-shooting mechanics (target barrels and gates)
- Collectible multiplier zones
- Simple auto-battle combat
- Progressive difficulty across 20+ stages

Think Temple Run meets auto-chess, with a medieval fantasy theme.

## The Tech Stack

**Why Godot 4?** 
- 100% free and open source
- Excellent 3D engine for mobile
- GDScript is perfect for AI-assisted development
- One-click Android export
- Zero hidden costs

**Development Approach:**
- Build-first learning (minimal tutorials)
- Using Claude Code for guidance and code generation
- Free assets from Kenney.nl
- Git for version control
- Obsidian vault for documentation

## Development Timeline

### Weeks 1-2: Core Prototype
Building the foundation: movement, lane switching, basic combat, and collectibles.

### Weeks 3-4: Advanced Mechanics  
Auto-shooting system, gate mechanics, multiplier zones, particle effects.

### Weeks 5-8: Content & Polish
Multiple unit types, enemy variety, visual effects, sound design.

### Weeks 9-12: Stage Creation
Creating 20+ unique stages with progressive difficulty.

### Weeks 13-16: Optimization
Android performance tuning, touch controls, screen adaptation.

### Weeks 17-20: Final Polish
Testing, refinement, and APK preparation.

## Why This Project?

**Learning Goals:**
- Master Godot 4's 3D capabilities
- Understand mobile game optimization
- Learn Android export process
- Practice AI-assisted development with Claude Code

**Design Philosophy:**
> "Build first, learn as you go. You learn better by doing than by watching tutorials."

This approach means diving into actual game development immediately, using Claude Code to help navigate challenges as they arise.
